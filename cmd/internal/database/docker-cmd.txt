1. Setup and run pgx container via Docker
docker run --name <container-name> -e POSTGRES_PASSWORD=<password> -p 5431:5432 -d postgres

-e : Environment Variable
-p : Port Mapping
-d : Detached
postgres : Image to used for this container

2. List running containers
docker ps or docker ps -a

-a : List all containers

3. TablePlus configs:
Name:         <container-name> (or whatever you prefer)
Connection:   PostgreSQL
Host:         localhost
Port:         5431
User:         postgres
Password:     <password>
Database:     postgres

4. psql docker connect (Terminal Access)
docker exec -it  <container-name> psql -U postgres
or
psql "connection-string"

-it : Interactive terminal session
-U : User

5. Stop pgx container
docker stop <container-name>

6. Delete pgx container
docker rm <container-name>

(Use migrations to set tables back up)

7. Persisting Volumes
docker voume create <container-name + "-data">
docker run --name <container-name> -e POSTGRES_PASSWORD=<password> -p 5431:5432 -v <container-name + "-data">:<filepath> -d postgres


While resetting containers one loses all the data within that container to deal with that, to persist that use volumes to save data to disk

8. Connection string:
protocol://username:password@host:port/database

protocol: postgres
